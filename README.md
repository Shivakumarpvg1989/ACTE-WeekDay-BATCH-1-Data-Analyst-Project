# ACTE WeekDay BATCH 1 Data Analyst Portfolio Project - Pizza Sales Analysis
Data Analyst Project Source code Raw Dataset and Dashboard SQL and Notebooks 

The Pizza Sales Analysis project focuses on analyzing transactional sales data to uncover business insights, improve decision-making, and identify revenue growth opportunities. The dataset includes order details such as pizza types, categories, sizes, quantities, prices, and timestamps.

🎯 Objective

The main goal of this project is to:

Analyze overall sales performance
Identify top-selling and low-performing pizzas
Understand customer ordering patterns
Provide actionable insights to improve business revenue
📊 Key Business Questions Solved
What is the total revenue, total orders, and total pizzas sold?
Which pizzas generate the highest and lowest revenue?
What are the peak order days and times?
Which pizza category and size contribute most to sales?
What are the top 5 best-selling pizzas?
🛠️ Tools & Technologies Used
SQL – Data extraction and aggregation
Python (Pandas, Matplotlib) – Data cleaning and visualization
Excel – Initial data exploration
Power BI – Interactive dashboard creation
📈 Key Insights
The business generated significant revenue from large-sized pizzas, contributing the highest share.
Weekends (Friday & Saturday) recorded peak sales.
Certain pizzas consistently performed as top sellers, contributing major revenue.
Some pizzas had very low sales, indicating opportunities for menu optimization.
Afternoon and evening hours showed the highest order volume.
📊 Dashboard Highlights (Power BI)
KPI cards: Total Revenue, Total Orders, Total Pizzas Sold
Bar charts: Revenue by Pizza Category & Size
Line charts: Daily & Monthly Sales Trends
Pie charts: Category-wise contribution
Top/Bottom 5 pizzas visualization
💡 Business Recommendations
Focus marketing campaigns on high-performing pizzas
Introduce offers during low-sales hours
Optimize menu by removing or improving low-performing items
Increase stock and staffing during peak hours
